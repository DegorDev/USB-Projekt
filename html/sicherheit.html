Sicherheit

USB-Sticks sind eine sehr beliebte Möglichkeit für einen schnellen und unkomplizierten Datenaustausch. Doch die wenigsten Nutzer wissen, dass auch ein USB Stick ein hohes Sicherheitsrisiko darstellen kann.

Auf den beiden IT-Messen Def Con und BlackHat wurde von Sicherheitsexperten auf ein Horrorszenario hingewiesen, welches sehr realistisch ist.

Eine mögliche Angriffsstrategie zielt auf die Umprogrammierung des USB-Controller-Chips ab. Diese sind sehr weit verbreitet und kommen z.B. in USB-Sticks zum Einsatz. Einen wirksamen Schutz gegen diese Umprogrammierung gibt es derzeit nicht. Viele Hersteller verzichten auf Schutzmaßnamen diesbezüglich. So kann also jeder USB-Stick potenziell schädlich sein.

Mögliche Szenarien die eine solche Umprogrammierung nach sich ziehen kann, wären folgende:

  • USB-Sticks können so umprogrammiert werden, dass sich der USB-Stick z.B. als Tastatur ausgibt und so Eingaben des Nutzers emulieren kann um dann z.B. Maleware auf dem PC zu installieren.
  • Der USB-Stick gibt sich als Netzwerkkarte aus und kann so die DNS-Einstellungen des Computers ändern, um dann den gesamten Datenverkehr des Computers um zu leiten.
  • Modifizierte USB-Sticks können beim Bootvorgang einen Virus laden, der das Betriebssystem noch vor dem Booten infiziert.

Da die meisten Maleware-Scanner keinen Zugriff auf die Firmware von USB-Geräten haben, gibt es leider keinen effektiven Schutz gegen solche Angriffe. USB-Firewalls, welche nur bestimmte Geräteklassen blockieren können, existieren (noch) nicht. Einen gewissen Schutz bietet die Eigenschaft von macOS, den Benutzer beim Anstecken eines als Tastatur erkannten USB-Geräts zu fragen, ob die Tastatur aktiviert werden soll. Der effektivste und beste Schutz ist aber der, keine USB-Geräte zu verwenden von denen man nicht weiß wo sie herkommen (z.B. ein gefundener USB-Stick in der Schule, bei dem man nicht weiß wem er gehört). Aber einen 100% Schutz gegen Attacken über USB-Geräte gibt es nicht, außer man verzichtet Komplett auf USB-Geräte was in der heutigen Zeit eher schwer ist.

Eine weitere interessante Angriffsmöglichkeit war es, dass ein USB-Laufwerk über die AutoRun-Funktion einen Rechner Infiziert. Wie war das möglich? USB-Laufwerke waren in der Lage, auf die AutoRun-Funktion von Windows zu zugreifen, um selbstständig Programme zu starten. Das konnten sie, um z.B. Treiber für USB-Geräte zu installieren ohne das der Nutzer eine Installation manuell starten muss. Über diese Funktion war es aber auch möglich, dass der USB-Stick Schadsoftware (Maleware) auf dem Rechner installiert. Diese Sicherheitslücke wurde allerdings mit Windows 7 entfernt, als man USB-Laufwerken den Zugriff auf die AutoRun-Funktion verwehrte. Rechner mit älteren Windows Versionen (z.B. Windows XP oder Windows Vista) können allerdings immer noch über diese Methode angegriffen werden.
